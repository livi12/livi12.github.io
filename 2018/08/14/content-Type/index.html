content-Type 三种格式的区别
Post：2018-08-14 15:16:09 Visit：

content-type 表明信息的类型，缺省值为” text/plain”。
主要类型有9种，分别是application、audio、example、image、message、model、multipart、text、video。常见的有以下几种。

text/plain：纯文本，文件扩展名.txt
text/html：HTML文本，文件扩展名.htm和.html
image/jpeg：jpeg格式的图片，文件扩展名.jpg
image/gif：GIF格式的图片，文件扩展名.gif
audio/x-wave：WAVE格式的音频，文件扩展名.wav
audio/mpeg：MP3格式的音频，文件扩展名.mp3
video/mpeg：MPEG格式的视频，文件扩展名.mpg
application/zip：PK-ZIP格式的压缩文件，文件扩展名.zip

http 常见的提交数据的方式 application/x-www-form-urlencoded，multipart/form-data，application/json

application/x-www-form-urlencoded

如果不设置 enctype 属性，那么最终就会以 application/x-www-form-urlencoded 方式提交数据。这种方式提交的数据，会将数据中的空格转化为+ ，特殊字符被编码为ASCII十六进制字符

提交的数据格式 按key1=val1&key2=val2 的方式进行编码，key 和 val 都进行了 URL 转码 如下,序列话的对象以 & 连接。

form 的 enctype 属性可以设置
application/x-www-form-urlencoded 对字符进行编码
multipart/form-data 不对字符编码
text/plain 空格转换为 “+” 加号，但不对特殊字符编码。

multipart/form-data

设置

表单的 enctype 为multipart/form-data，也是一种常见的表单提交方式 ，查看请求的 request Headers,包含文件上传控件的表单时，必须使用该值

boundary 用于分割不同的字段，内容部分在 request payload 中显示

  • from 表单传输文件，需要设置 enctype=”multipart/form-data”
  • 不显示用from 表单传输文件，可以用fromData 传输,formData 对象用以将数据编译成键值对，以便用XMLHttpRequest来发送数据。其主要用于发送表单数据，但亦可用于发送带键数据(keyed data)，而独立于表单使用。传输的数据和form 表单的submit提交的数据格式一样， formData 有兼容性问题，根据需要使用

post 上传文件，不能设置content-Type

application/json

设置application/json 这个 Content-Type 作为响应头，用来告诉服务端消息主体是序列化后的 JSON 字符串。
此处的序列化与 application/x-www-form-urlencoded 序列化不同，application/json 的序列化 用JSON.stringify 即可用，参数格式

根据数据后台接收的数据格式，对数据进行转化

扩展

一个html form 可以已四种方式提交

  • POST 方法，设置enctype 属性 application/x-www-form-urlencoded (default)
  • POST 方法，设置enctype 属性 text/plain
  • POst 方法，设置enctype 属性 multipart/form-data
  • GET 方法，enctype 属性会被忽略。
